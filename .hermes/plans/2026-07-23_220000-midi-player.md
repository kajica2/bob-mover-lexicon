# MIDI Player Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Add a built-in MIDI player to the Practice page that plays the exercise's notes as audio, with synchronized score highlighting, adjustable tempo, and optional metronome click.

**Architecture:** Pure-vanilla Web Audio API (no external libs). A small `playbackEngine.js` module (loaded as a separate script from `practice/practice.js`) wraps `AudioContext` + `OscillatorNode` per note + ADSR envelope + a metronome tick. The Practice page renders play/stop/tempo/metronome controls and forwards start/stop commands. Notes are already parsed in `state.currentScore.notes` (MIDI pitch + beat + duration) by the existing `parseMusicXML()`; the player reads them directly. Verovio's SVG output has `id` attributes on notes (`note-NNN`) which we use to highlight the currently-playing note.

**Tech Stack:** Web Audio API (already in scope — used by the audio recorder), vanilla JS/CSS. No npm dependencies, no new CDN scripts.

## Current Context / Assumptions

- **Existing audio**: `practice/practice.js` already imports Web Audio for recording (`AudioContext` for the live waveform analyser). A comment at line 839 says "Playback synth was removed; create a transient one just for the analyser/meter." So we are reintroducing playback.
- **Notes already parsed**: `parseMusicXML()` (line 33-84) walks the served MusicXML, producing `state.currentScore = { bpm, notes: [{midi, beat, duration}, ...], keyFifths }`. Notes are in beats; bpm is read from `<sound tempo>` or defaults to 120.
- **Cycle mode**: when the user clicks Apply with a non-Off cycle, the served XML is extended; `renderScore()` re-parses notes. The player should pick up the latest `state.currentScore` each time Play is hit (no need to track cycle state separately).
- **Instrument clamping**: server-side `clamp_notes_to_range()` already transposes out-of-range notes an octave at serve time. The player plays whatever the served XML has — no client-side range work needed.
- **Etudes**: etudes bypass `/api/musicxml` and load MusicXML directly from IndexedDB via `loadEtude()`. They still pass through `renderScore(xml)` which calls `parseMusicXML()` and sets `state.currentScore`. So the player works for etudes too — no special-casing.
- **Range modal**: the AudioContext can't be created during the first-visit range modal (the modal blocks everything visually but JS still runs; it's fine to lazily init the AudioContext when Play is clicked).
- **README drift**: README.md lines 21-22 and 51 describe a Web Audio API player that no longer exists. Task 8 updates README to match reality.

```javascript
/* practice/playback-engine.js
 * MIDI playback engine for the Practice page.
 *
 * Public API (window.playbackEngine):
 *   init() -> boolean           ensure AudioContext is ready; user-gesture required
 *   setNotes(notes, bpm)        schedule notes from [{midi, beat, duration}], tempo
 *   play({ onNote, onBeat, onEnd })  start playback, returns immediately
 *   stop()                       halt + cancel scheduled events
 *   setTempo(bpm)                adjust tempo on the fly (recomputes schedules)
 *   setMetronome(on)             toggle a click on every beat
 *   dispose()                    close AudioContext, free resources
 *
 * Notes are in beats (relative to start); bpm is integer-ish. We schedule
 * against AudioContext.currentTime.
 */
(function (root) {
  const ADSR = { attack: 0.01, decay: 0.08, sustain: 0.55, release: 0.12 };
  const MASTER_GAIN_PEAK = 0.18;   // gentle — Web Audio feedback is harsh

  const engine = {
    ctx: null,
    master: null,
    notes: [],   // [{midi, beat, duration}]
    bpm: 120,
    startTime: 0,  // ctx.currentTime when playback started
    scheduled: [], // {osc, gain, endTime} for cleanup
    metronomeOn: false,
    isPlaying: false,
    rafId: 0,

    _ensureCtx() {
      if (this.ctx) return true;
      try {
        const Ctor = window.AudioContext || window.webkitAudioContext;
        if (!Ctor) return false;
        this.ctx = new Ctor();
        this.master = this.ctx.createGain();
        this.master.gain.value = MASTER_GAIN_PEAK;
        this.master.connect(this.ctx.destination);
      } catch (e) {
        console.warn('AudioContext failed to initialize:', e);
        return false;
      }
      return true;
    },

    init() {
      if (!this._ensureCtx()) return false;
      if (this.ctx.state === 'suspended') {
        // Modern browsers require a user gesture; we'll attempt to resume
        // here and let the caller retry on actual click if still suspended.
        return this.ctx.resume();
      }
      return true;
    },

    setNotes(notes, bpm) {
      this.stop();
      this.notes = Array.isArray(notes) ? notes.slice() : [];
      this.bpm = Number(bpm) > 0 ? Number(bpm) : 120;
    },

    setTempo(bpm) {
      const newBpm = Number(bpm) > 0 ? Number(bpm) : this.bpm;
      if (this.isPlaying && newBpm !== this.bpm) {
        // Re-anchor: figure out elapsed beats, restart with new tempo.
        const ctxNow = this.ctx.currentTime;
        const elapsedBeats = (ctxNow - this.startTime) * (this.bpm / 60);
        this.bpm = newBpm;
        this.stop(/* keepProgress */ false);
        this._scheduleFromOffset(elapsedBeats);
        this._resumeRaf();
      } else {
        this.bpm = newBpm;
      }
    },

    setMetronome(on) { this.metronomeOn = !!on; },

    play({ onNote, onBeat, onEnd } = {}) {
      if (!this._ensureCtx()) return false;
      if (this.ctx.state === 'suspended') {
        // Will resolve on the next user click; ignore for now.
        return false;
      }
      this.stop();
      this.startTime = this.ctx.currentTime + 0.05; // tiny lead-in
      this._scheduleFromOffset(0, { onNote, onBeat });
      this.isPlaying = true;
      this._scheduleEnd(onEnd);
      this._resumeRaf();
      return true;
    },

    _scheduleFromOffset(offsetBeats, { onNote, onBeat } = {}) {
      const secondsPerBeat = 60 / this.bpm;
      const startOffset = this.startTime + offsetBeats * secondsPerBeat;
      // Schedule notes.
      for (const n of this.notes) {
        const noteStart = this.startTime + n.beat * secondsPerBeat;
        if (noteStart < startOffset) continue;
        const noteEnd = noteStart + Math.max(0.02, n.duration * secondsPerBeat);
        this._scheduleNote(n.midi, noteStart, noteEnd, () => {
          if (onNote) onNote(n);
        });
      }
      // Schedule metronome clicks if enabled.
      if (this.metronomeOn) {
        const totalBeats = this.notes.reduce(
          (m, n) => Math.max(m, n.beat + n.duration), 0
        );
        const beatCount = Math.ceil(totalBeats);
        for (let i = 0; i <= beatCount; i++) {
          const t = this.startTime + i * secondsPerBeat;
          if (t < startOffset) continue;
          const isDownbeat = (i % 4 === 0);
          this._scheduleClick(t, isDownbeat);
          const fireBeat = (i * secondsPerBeat) + (this.startTime - this.ctx.currentTime);
          // Schedule the onBeat callback to fire visually at the right time.
          this._scheduleCallback(t, () => { if (onBeat) onBeat(i); });
        }
      }
    },

    _scheduleNote(midi, startTime, endTime, onNoteStart) {
      const freq = 440 * Math.pow(2, (midi - 69) / 12);
      const osc = this.ctx.createOscillator();
      const gain = this.ctx.createGain();
      // Triangle wave gives a vaguely sax-like tone; well within Web Audio defaults.
      osc.type = 'triangle';
      osc.frequency.setValueAtTime(freq, startTime);
      // ADSR envelope so notes don't click.
      const peak = 1.0;
      const attackEnd = startTime + ADSR.attack;
      const decayEnd = attackEnd + ADSR.decay;
      const releaseStart = Math.max(startTime + 0.04, endTime - ADSR.release);
      gain.gain.setValueAtTime(0, startTime);
      gain.gain.linearRampToValueAtTime(peak, attackEnd);
      gain.gain.linearRampToValueAtTime(peak * ADSR.sustain, decayEnd);
      gain.gain.setValueAtTime(peak * ADSR.sustain, releaseStart);
      gain.gain.linearRampToValueAtTime(0, endTime);
      osc.connect(gain);
      gain.connect(this.master);
      osc.start(startTime);
      osc.stop(endTime + 0.05);
      // Fire onNoteStart on the next audio quantum after startTime so the
      // visual highlight is in sync with audible attack.
      this._scheduleCallback(startTime, onNoteStart);
      this.scheduled.push({ osc, gain, endTime });
      osc.onended = () => {
        try { osc.disconnect(); gain.disconnect(); } catch (e) {}
      };
    },

    _scheduleClick(time, isDownbeat) {
      const osc = this.ctx.createOscillator();
      const gain = this.ctx.createGain();
      osc.type = 'square';
      osc.frequency.setValueAtTime(isDownbeat ? 1600 : 1000, time);
      gain.gain.setValueAtTime(0, time);
      gain.gain.linearRampToValueAtTime(isDownbeat ? 0.10 : 0.05, time + 0.005);
      gain.gain.linearRampToValueAtTime(0, time + 0.05);
      osc.connect(gain);
      gain.connect(this.master);
      osc.start(time);
      osc.stop(time + 0.06);
      osc.onended = () => {
        try { osc.disconnect(); gain.disconnect(); } catch (e) {}
      };
    },

    _scheduleCallback(time, fn) {
      // Use a setTimeout that's clamped to the audio clock; small drift is OK
      // because we already budget a 0.05s lead-in.
      const delayMs = Math.max(0, (time - this.ctx.currentTime) * 1000);
      const id = setTimeout(fn, delayMs);
      this.scheduled.push({ type: 'callback', id });
    },

    _scheduleEnd(onEnd) {
      const totalBeats = this.notes.reduce(
        (m, n) => Math.max(m, n.beat + n.duration), 0
      );
      const endSeconds = totalBeats * (60 / this.bpm) + 0.2;
      this._scheduleCallback(this.startTime + endSeconds, () => {
        this.isPlaying = false;
        if (onEnd) onEnd();
      });
    },

    stop() {
      // Cancel oscillators
      for (const s of this.scheduled) {
        if (s.type === 'callback') {
          clearTimeout(s.id);
        } else if (s.osc) {
          try { s.osc.stop(0); } catch (e) {}
          try { s.osc.disconnect(); } catch (e) {}
          try { s.gain.disconnect(); } catch (e) {}
        }
      }
      this.scheduled = [];
      this.isPlaying = false;
      if (this.rafId) {
        cancelAnimationFrame(this.rafId);
        this.rafId = 0;
      }
    },

    _resumeRaf() {
      // RAF loop is optional but useful if we later want continuous cursor
      // animation; for now, we cancel cleanly.
      if (this.rafId) cancelAnimationFrame(this.rafId);
      this.rafId = 0;
    },

    dispose() {
      this.stop();
      if (this.ctx && this.ctx.state !== 'closed') {
        try { this.master.disconnect(); } catch (e) {}
        this.ctx.close();
      }
      this.ctx = null;
      this.master = null;
    },
  };

  root.playbackEngine = engine;
})(window);
```

```javascript
// ===== MIDI playback =====
// Uses the playbackEngine loaded from playback-engine.js. The engine is
// initialized lazily on first play (browsers require a user gesture).
function wirePlaybackControls() {
  const playBtn = document.getElementById('btn-playback-play');
  const stopBtn = document.getElementById('btn-playback-stop');
  const tempoSlider = document.getElementById('playback-tempo');
  const tempoLabel = document.getElementById('playback-tempo-label');
  const metroBtn = document.getElementById('btn-playback-metronome');
  const statusEl = document.getElementById('playback-status');

  if (!playBtn || !stopBtn) return; // not on this page

  function setStatus(text, cls) {
    if (!statusEl) return;
    statusEl.textContent = text;
    statusEl.className = 'playback-status' + (cls ? ' ' + cls : '');
  }

  // The tempo slider reports a percentage of the exercise's written tempo.
  function refreshTempo() {
    if (!tempoSlider || !state.currentScore) return;
    const pct = parseInt(tempoSlider.value, 10) || 100;
    tempoLabel.textContent = pct + '%';
    const bpm = Math.max(20, Math.round((state.currentScore.bpm || 120) * pct / 100));
    window.playbackEngine.setTempo(bpm);
  }

  playBtn.addEventListener('click', async () => {
    if (!state.currentScore || !state.currentScore.notes || !state.currentScore.notes.length) {
      setStatus('no notes to play', 'error');
      return;
    }
    const ok = window.playbackEngine.init();
    if (!ok && window.playbackEngine.ctx && window.playbackEngine.ctx.state === 'suspended') {
      // Browser blocked until user-gesture; we already are one but iOS
      // sometimes needs a second tap. Retry on the next click.
      try { await window.playbackEngine.ctx.resume(); } catch (e) {}
    }
    const pct = parseInt(tempoSlider.value, 10) || 100;
    const bpm = Math.max(20, Math.round((state.currentScore.bpm || 120) * pct / 100));
    window.playbackEngine.setNotes(state.currentScore.notes, state.currentScore.bpm || 120);
    window.playbackEngine.setTempo(bpm);
    const started = window.playbackEngine.play({
      onNote: (n) => highlightNote(n, true),
      onEnd: () => {
        playBtn.disabled = false;
        stopBtn.disabled = true;
        clearNoteHighlights();
        setStatus('done', '');
      },
    });
    if (!started) {
      setStatus('audio unavailable', 'error');
      return;
    }
    playBtn.disabled = true;
    stopBtn.disabled = false;
    setStatus('playing', 'playing');
  });

  stopBtn.addEventListener('click', () => {
    window.playbackEngine.stop();
    playBtn.disabled = false;
    stopBtn.disabled = true;
    clearNoteHighlights();
    setStatus('ready');
  });

  if (tempoSlider) {
    tempoSlider.addEventListener('input', refreshTempo);
  }

  if (metroBtn) {
    metroBtn.addEventListener('click', () => {
      const on = !metroBtn.classList.contains('active');
      metroBtn.classList.toggle('active', on);
      metroBtn.setAttribute('aria-pressed', on ? 'true' : 'false');
      window.playbackEngine.setMetronome(on);
    });
    // Initial pressed state
    metroBtn.setAttribute('aria-pressed', 'false');
  }
}
```

### Task 4: Synchronized score highlight

**Objective:** As each note plays, highlight the corresponding note in the Verovio-rendered SVG so the user can follow along visually.

**Files:**
- Modify: `practice/practice.js`

**Background:** Verovio's SVG output assigns an `id` attribute to each `<g class="note">` element. The id is something like `note-0000001234567890` (a numeric timestamp). We can match by **horizontal position** rather than by id, since id values aren't deterministic across renders.

**Step 4.1:** Implement `highlightNote(note, on)` and `clearNoteHighlights()`.

```javascript
function clearNoteHighlights() {
  if (!document.getElementById('score-container')) return;
  // Remove the highlight class from any previously active notes.
  document.querySelectorAll('#score-container g.note.active')
    .forEach(el => el.classList.remove('active'));
}

function highlightNote(note, on) {
  // Find all <g class="note"> elements in the rendered SVG. Pick the one
  // whose bounding-box left edge is closest to the expected position.
  // Cheaper approach: precompute per-beat x-positions when the score is
  // first rendered, then look them up.
  if (!note) return;
  const positions = state.scoreNotePositions; // [{x: px, beat: number}, ...]
  if (!positions) return;
  // Find the position with closest beat to this note's beat.
  let best = positions[0], bestDiff = Infinity;
  for (const p of positions) {
    const d = Math.abs(p.beat - note.beat);
    if (d < bestDiff) { bestDiff = d; best = p; }
  }
  if (!best || !best.el) return;
  if (on) {
    best.el.classList.add('active');
  } else {
    best.el.classList.remove('active');
  }
}
```

```javascript
function indexScoreNotePositions(score) {
  if (!score || !score.notes || !score.notes.length) return [];
  const noteGroups = Array.from(document.querySelectorAll('#score-container g.note'));
  if (!noteGroups.length) return [];
  // Build (x, noteIndex) pairs: each note in score.notes corresponds to one
  // <g class="note"> in the SVG in document order. We sort by X to find the
  // matching beat.
  const positions = noteGroups.map((el, i) => {
    const bb = el.getBBox ? el.getBBox() : { x: 0 };
    return { x: bb.x, beat: score.notes[i] ? score.notes[i].beat : 0, el };
  });
  positions.sort((a, b) => a.x - b.x);
  return positions;
}
```

```javascript
async function loadExercise(id) {
  // Cancel any in-flight playback so notes from the previous exercise don't
  // continue sounding under the new score.
  if (window.playbackEngine) {
    window.playbackEngine.stop();
```

## Risks, Tradeoffs, Open Questions

1. **Audio quality**: a triangle wave is serviceable for ear-training but harsh for sustained listening. A future iteration could swap in Tone.js or a SoundFont (e.g., saxophone sample). v1 deliberately avoids the dependency cost.

2. **Verovio note ID stability**: I propose matching notes by horizontal X position rather than Verovio's `id` attribute, because IDs are timestamp-based and not deterministic across re-renders. If Verovio's id becomes deterministic in a future release, switching to id-based matching would be a small win.

3. **First-play latency**: on first click Play, the AudioContext may need to be (re)created or resumed. The player surfaces "audio unavailable" in that case. On iOS this might require a second tap; we'll know from the live matrix test.

4. **Tempo range**: 40-240% is wide but the slider step is 1%, which is fine. If users complain about granularity, easy to bump.

5. **No loop region yet**: jazz practice often wants to loop a 2-bar phrase. Out of scope for v1. Tracked as future work.

6. **SoundFont migration path**: if a user later wants "real sax", the swap is contained to `playback-engine.js`. The `setNotes` / `play` / `stop` API stays the same; only the oscillator code changes.
